1. 寄存器

CPU 中主要的部件是寄存器，8086 CPU 中有 14 个寄存器，本章我们要学习的是 AX、BX、CX、DX、CS、IP。其中前四个用来存放数据，后两个用于寻址。那 AX 来说，它是一个字，有 2 个 Byte，因此又分为 AH 和 AL。

寄存器和内存

内存是独立于 CPU 之外的硬件，CPU 通过指令和内存做数据交互，内存对 CPU 来说就是一个很大的一维线性空间，通过 CS & IP 来找到内存中对应的地址。

1.1 mov & add

注意事项：

  • 对于 8 位运算，al 产生的进位并不会存在 ah 中，因为 CPU 在执行这两条指令认为 al 和 ah 是两个不相关的寄存器。
  • 两个操作的对象位数应该保持一致，比如 mov ax,bl 就是错误的指令。

1.2 地址表示

8086 CPU 是 16 位的 CPU，但是它有 20 位的地址总线。其采用了两个 16 位地址合成方法来形成一个 20 位的物理地址。
$$
段地址*16+偏移地址=物理地址
$$
这里需要知道的是，地址是在内存上的地址，CPU 找到内存上对应的地址，并读取上面的数据。

“数据在 21F60H 中”对于 8086 的说法：数据存在内存 2000：1F60中，前者就是段地址，后者是偏移地址。

给定段地址，由于偏移地址有 16 位，因此最多能达到 $2^{16}$=64KB 的寻址能力。

LAB汇总

Lab1 使用Debug

2023.6.2

Debug 是 DOS、Windows 都提供的调试工具。使用它可以查看 CPU 各种寄存器中的内容，内存情况等。Debug 总共有二十多条命令，但是常用的只有六条。

环境配置：DosBox安装 ，感谢博主提供的教程和打包好的文件！配置起来很轻松。

六条最基本最常用的指令：

  • R：查看，改变 CPU 寄存器中的内容。
  • D：查看内容中的内容。
  • E：改写内存中的内容。
  • U：将内存中的机器指令翻译成汇编指令。
  • T：执行一条机器指令。
  • A：以汇编指令的格式在内存中写入一条机器指令。

实验任务：见书 P45

  1. 照着做就行。
  2. 给了一个计算一个数乘以 $2^n$ 的方法，就是不断地 mov ax,ax 循环 n 次。
  3. PC 机主板上的 ROM 有生产日期，在 FFF00H ~ FFFFFH 处，但是无法改变，因为这一段 ROM 是只读的，无法修改。
  4. B8100H 位于显存地址空间，用于屏幕的显示，类似于 Linux 中的 FrameBuffer。如果修改显存地址空间中的代码，屏幕上有些位置会出现奇特的图标。
